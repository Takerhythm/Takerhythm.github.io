发表于 2019-03-23 | 分类于 Django

记录一次django项目中的jwt认证bug,避免他人踩坑
该bug表现为登录成功后跳转回登录界面,如下图

断点调试

首先我要确定这是前段还是后端的问题

前端断点



最后发现跳转是下面代码的问题,因为后端返回403,所以页面又跳转回登录页面

接下来判断为什么出现403状态码,有三种可能
1.前端未带上jwt
2.前段带上错误的jwt
3.后端jwt配置错误
经过刚才的前端断点,第一第二条可以排除
所以我去配置文件中查看jwt配置

发现配置无误,陷入了沉思

后端断点

随后在后端进行断点调试,发现返回的是匿名用户身份
也就是说没走jwt认证
不知道全局配置为何不生效
